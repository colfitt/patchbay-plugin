Source-class contract (locked, identical to Plan 02):

```python
def match_url(url: str) -> bool: ...
def fetch_tier1(url: str) -> dict: ...
def parse_to_chunks(fetch_result: dict, gear_ctx: dict) -> list[dict]: ...
```

Chunk types this source class emits (from `skills/patchbay-ingest/references/chunk-schema.md`):
- `artist_usage`: content is `{artist, artist_roles, associated_act, verification_type, verification_note, verbatim_quote, summary, alternatives_recommended}`
- `cross_ref`: content is `{from_gear, relation, to_gear, weight}` where relation in {`used_with`, `similar_in_category`}
- `text` (for description/overview): content is markdown string
- `external_resource`: content is `{resource_type, creator, title, url, updated, relevance, citing_chunk_ids}` — populated when YouTube reviewer URL appears in `verbatim_quote`

Equipboard page DOM landmarks (from spike findings — `web-scraping.md` and spike 003b):
- Artist usage blocks: each artist appears as a section heading or `<div class="artist-usage">` (variable; selectors must be defensive)
- Used-with category rollup: a list block titled "Used With" containing other gear items
- Similar-in-category: a numbered top-10 list, often titled "Similar in [Category]"
- Critic reviews: blocks with quotes, each crediting a reviewer + outlet

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user URL input → match_url | Untrusted URL string enters source-class routing. |
| equipboard.com HTML response → parse_to_chunks | Untrusted HTML body parsed with BeautifulSoup. |
| extracted text → chunks.jsonl | Free-form artist quotes / gear names become chunk content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | Spoofing (URL scheme abuse) | `match_url` | mitigate | Reject non-http(s) schemes BEFORE host check. |
| T-03-14 | Spoofing (host substring confusion: `equipboard.com.attacker.io`) | `match_url` | mitigate | Exact-host set membership; no substring match. |
| T-03-15 | Tampering (XXE / external entity in HTML payload) | `parse_to_chunks` | mitigate | Use BeautifulSoup with `html.parser` (stdlib, no external XML processor). lxml deliberately avoided. |
| T-03-16 | Tampering (injection via free-form artist/gear text) | `parse_to_chunks` → `write_chunks` | mitigate | Extracted text is treated as opaque string data. All chunk objects pass through `json.dumps` in Plan 01's writer — newlines, quotes, control characters are JSON-escaped. No shell/eval evaluation. |
| T-03-17 | Information Disclosure (non-http URL embedded in artist block) | external URL extractor | mitigate | After regex match, validate via `urlparse` and discard non-http(s) schemes. |
| T-03-18 | Denial of Service (HTML bomb / billion-laughs equivalent) | `parse_to_chunks` | accept | BeautifulSoup with `html.parser` is not vulnerable to XML billion-laughs. HTML parser bounded by browser-like tolerance; document size is bounded by tier-1's 15s timeout (Plan 01). |

All high-severity threats mitigated. T-03-18 accepted given parser choice + upstream timeout.
</threat_model>

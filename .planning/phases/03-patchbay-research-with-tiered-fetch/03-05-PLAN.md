<objective>
Ship the user-driven escalation flow that wraps the cheap-by-default tier-1 spine built in Plan 01. Purpose: This is the load-bearing UX of the cheap-by-default + user-driven escalation model. Without it, tier-1 failures are dead-letter entries.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| failures.log → review loop | Parsed JSONL becomes UI state — malformed lines must not crash the loop. |
| user choice → escalation dispatcher | User-typed choice routes to one of four execution paths. |
| MCP tool surface → review loop | Tier-2 and tier-3 MCP responses are untrusted (return values from external tools). |
| pasted DOM (tier 0) → parser | User-pasted body is untrusted free-form text. |
| tier-2 / tier-3 fetched body → parser → chunks.jsonl | Same boundary as Plans 02-04; mitigations inherited. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-27 | Tampering (malformed failures.log line crashes review loop) | `load_failures` | mitigate | Wrap each `json.loads` in try/except; print warning, skip line, continue. Never raise to caller. |
| T-03-28 | Elevation of Privilege (RCE via crafted URL in tier-3 `open` invocation) | `tier3_vision.fetch_tier3` | mitigate | `subprocess.run(argv=["open", "-na", "Google Chrome", "--args", "--new-window", url], shell=False)`. URL is one argv element. Re-validate URL via `urlparse` before calling — reject non-http(s). |
| T-03-29 | Spoofing (fake `list_connected_browsers` response) | `tier2_chrome.precheck_chrome_extension` | accept | The MCP tool surface is trusted infrastructure (Claude's own tool layer). We use the returned value as a boolean signal only — no privilege grant downstream depends on the contents. |
| T-03-30 | Tampering (malicious pasted body in tier 0) | `review_failures` paste branch | mitigate | Pasted body is treated as opaque string passed to the matched source class's `parse_to_chunks`.

<objective>
Close the citation loop: let the user act on a recommendation surfaced by Plan 04-02 by marking it verified. The verify flow (a) canonicalizes the input URL, (b) confirms an existing external_resource chunk represents it, (c) dispatches downstream ingestion via the existing url_router (Plan 03-01) — tier-1 for articles, multimodal pipeline for YouTube — (d) stamps every produced chunk with `trust: "high"` at the chunk-dict level, and (e) flips the external_resource chunk's relevance to "verified" and adds its own `trust: "high"` flag via the atomic update_chunk_field helper.

Purpose: CITATION-03 says "User can mark a surfaced resource as verified → triggers ingestion → promotes the resulting chunks to high-trust in the knowledge store." The "high-trust" promotion is a chunk-level flag (per planning context: "field/flag on the resulting chunks, not a separate file"). This plan is the last brick in the Phase 4 loop — once it ships, the user has the full Plan 02 → Plan 03 workflow.

Output:
- verify_resource.py — orchestrator (lookup → dispatch → promote → flag), invocable via CLI.
- write_chunk.py — minimal additive change: append a new keyword-only `trust=None` parameter to write_chunks AFTER the existing `gear_root` kwarg; stamps every chunk pre-append when set.
- A reference doc + SKILL.md additive update.
</objective>

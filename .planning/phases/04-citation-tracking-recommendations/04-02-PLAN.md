<interfaces>
External_resource chunk shape (guaranteed by Plan 04-01's post-write sweep):

  - id: "ext-..."  (sweep-emitted IDs use the stable form `ext-sweep-{sha1(canonical_url)[:8]}` — Plan 04-01)
  - type: "external_resource"
  - source: emitter source class name ("reddit" | "equipboard" | "youtube" | "sweep")
  - content:
    - resource_type: "youtube" | "article" | "reddit-post" | "image" | "other"
    - creator: str (may be "")
    - title: str (may be "")
    - url: canonicalize_url output
    - updated: str | None
    - relevance: str
    - citing_chunk_ids: list[str] — ids of chunks that reference this URL
  - tier_used: int | None  (None for sweep-emitted; 1/2/3 for fetched; Plan 04-01)
  - provenance: {url, scraped_at, ...}

Distinct-source rule for the N-threshold (CITATION-02 success criterion 3):
- The spec says "N independent sources reference the same canonicalized URL".
- INTERPRETATION: count the number of DISTINCT values of `citing_chunk["source"]`
  across the chunks named in `citing_chunk_ids`, NOT the length of `citing_chunk_ids`.
- The external_resource chunk's OWN `source` field is NOT counted — only the
  sources of its citing chunks.
- Rationale: a single Reddit thread (one source) emitting two external_resource
  references to the same YouTube video should NOT trip a threshold of 2. Two
  INDEPENDENT sources (e.g., reddit + equipboard) referencing the same video
  SHOULD trip it.
- Known limitation (v2.0): when same-class chunks dominate, the distinct-source
  count may under-represent true independence. Example: an Equipboard page that
  re-publishes a YouTube reviewer's transcript is ONE source ("equipboard") in
  the count, even though the underlying primary sources are two. Proper
  primary-source-independence tracking (a `primary_sources: list[str]` field on
  citing chunks capturing the upstream re-published sources) is deferred to a
  future phase. For v2.0, distinct citing-source count is the load-bearing
  observable; the precision floor is acceptable.

CLI contract:
- Invocation: `python3 skills/patchbay-research/scripts/citations.py <chunks_path> --gear "<Brand Item>"`
- Optional flags:
  - `--threshold N` (int >= 1, default 2)
  - `--filter-url <url>` (show only recommendations whose canonical URL matches)
  - `--json` (machine-readable output; default is markdown)
- Exit code: 0 on success (including the no-results case). Non-zero only on
  argument errors.

Recommendation dataclass shape (consumed by Plan 03):
- canonical_url: str
- resource_type: str
- creator: str
- title: str
- independent_source_count: int (the value compared against threshold)
- citing_chunks: list of dicts {id, source, excerpt}
- external_resource_chunk_id: str

Markdown output shape — first heading line for each recommendation:
  "## N. <canonical_url> — referenced X times across Y sources"

Empty result (threshold not crossed by any URL) prints a single guidance line
(LOCKED at v2.0 per W3 fix — does NOT suggest re-running /patchbay:research,
which would be misleading since the user has just done that to gather the
substrate the --citations command just queried):
  "No citation recommendations at threshold N=<threshold> for <gear>. Try --threshold 1 to see all external resources."

Excerpt rule: walk the citing chunk's `content` as strings (mirrors Plan 04-01's
sweep), find the first occurrence of a URL whose canonical form matches the
recommendation's canonical_url, and return ~80 chars surrounding it (with "..."
prefix/suffix if truncated). Fallback: first 80 chars of any content string.

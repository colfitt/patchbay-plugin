<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → SKILL invocation | User provides `<gear>` and optionally `<url>`. Both are untrusted strings entering the system. |
| internet → tier-1 fetch | Remote servers return arbitrary HTML/JSON bodies. Body content is untrusted; status codes are untrusted. |
| fetched bytes → chunks.jsonl | Parsed source content becomes lines in the per-gear knowledge store. |
| fetched bytes → failures.log | Failure metadata derived from response headers/body content. |
| user filesystem → skill output | Skill writes to `<gear_root>/<Brand Item>/knowledge/`. Gear arg must not escape gear_root. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure (SSRF) | `fetch_tier1.py` | mitigate | Reject non-http(s) schemes (`file://`, `ftp://`, etc.). Reject private/loopback IP destinations using `ipaddress` stdlib (127/8, 10/8, 192.168/16, 172.16/12, ::1, fc00::/7). Surface refusal in SKILL.md error-handling table. |
| T-03-02 | Tampering (HTML/JSON injection into parser) | `fetch_tier1.py` body handling | accept | BeautifulSoup is the standard tier-1 parser per spike findings. Tier-1 returns raw `body` string; source-class plans (02/03/04) own parse-side hardening (use `lxml`/`html.parser`, never `eval`-style parsing). Risk: tier-1 itself does not execute body content; it stores it. Accepted at tier-1 boundary, mitigated downstream. |
| T-03-03 | Tampering (JSONL injection into chunks.jsonl / failures.log) | `write_chunk.py`, `log_failure.py` | mitigate | Every line is the output of `json.dumps()` on a structured dict.

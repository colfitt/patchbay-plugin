External_resource chunk shape (CITATION-01):
```python
{
    "id": "ext-<canonical-host>-<NN>",      # stable, source-prefixed
    "type": "external_resource",
    "source": "<emitter-source>",            # the source class that introduced this URL; or "sweep" if backfilled
    "content": {
        "resource_type": "youtube" | "article" | "reddit-post" | "image" | "other",
        "creator": str,                       # may be "" if unknown
        "title": str,                         # may be "" if unknown
        "url": str,                           # CANONICALIZED form (canonicalize_url output)
        "updated": str | None,                # ISO date or None
        "relevance": str,                     # may be "" if unknown
        "citing_chunk_ids": [str, ...]        # ids of every NON-external_resource chunk referencing this URL
    },
    "tier_used": int | None,                  # tier of the chunk that introduced the URL; 0/1/2/3 for fetched chunks; NULL for sweep-emitted chunks (the sweep does not fetch — see "Tier semantics" below)
    "provenance": {
        "url": str,                           # canonical URL again (deep-link target)
        "scraped_at": "ISO-8601 UTC",
        # additional provenance fields are additive per chunk-schema.md
    }
}
```

Tier semantics for sweep-emitted external_resource chunks (locked at v2.0):
- The spike-findings tier ladder reserves `tier_used=0` for "manual user-paste (escape hatch)" — content the USER hand-fed into the system. A sweep-emitted external_resource is the OPPOSITE of that: it is a synthetic chunk derived from URLs already present in OTHER chunks; no fetch happened, no user paste happened.
- Therefore sweep-emitted external_resource chunks set `tier_used = None` (JSON `null`) to mean "not applicable, this chunk did not come from a fetch tier." This requires no schema doc change (the schema's `tier_used` field is documented as additive/optional shape and the sweep doc — `references/external-resource-sweep.md` — calls out the `null` value explicitly).
- Existing source-class-emitted external_resource chunks (reddit, equipboard) retain their fetched tier_used value verbatim — the sweep canonicalizes their URL and merges citing_chunk_ids but does NOT rewrite their tier_used.
- Downstream consumers (Phase 3 RESEARCH-01 queries, Plan 02 aggregator, Plan 03 verify) MUST treat `tier_used is None` as "did not fetch" — equivalent in queries to "filter only by tier_used in {1,2,3} when a fetched-source filter is desired." Plan 02's aggregate_citations does NOT filter by tier_used today (it groups by canonical URL), so this is forward-compatible. The Phase 3 RESEARCH-01 success criterion 1 query ("count chunks by tier_used") gracefully ignores null entries the same way it would ignore a missing key.

Resource-type override on merge (locked at v2.0):
- A `reddit.com/r/X/comments/Y` URL emitted by reddit.py carries `resource_type="article"` (reddit's `_classify_url` does not know the "reddit-post" type). The sweep's `_classify_resource_type` returns `"reddit-post"` for the same URL.
- Without an override, first-occurrence-wins on merge makes the final classification ordering-dependent (which source-class wrote it first determines the type).
- THE SWEEP IS AUTHORITATIVE: on every merge, the sweep OVERWRITES the existing chunk's `content.resource_type` with `_classify_resource_type(canonical_url)`. The sweep is the single source of truth for resource_type classification.
- This is acknowledged tech debt: a clean follow-up would extract a shared `classify_url` helper used by both reddit.py and the sweep so the override is unnecessary. For Phase 4 the sweep's override is the load-bearing fix; the helper extraction is a Phase-5+ refactor candidate.
- Task 2 test_sweep_overrides_existing_resource_type verifies this.

Sweep-emitted external_resource ID scheme (locked at v2.0):
- ID is STABLE across re-runs: `f"ext-sweep-{hashlib.sha1(canonical_url.encode('utf-8')).hexdigest()[:8]}"`.
- Rationale: a monotonic counter (`ext-sweep-001`, `ext-sweep-002`, ...) re-issues the same ID for a DIFFERENT canonical URL across runs when the discovery order changes (a new citing chunk lands earlier in the second run, bumping the counter assignment). A hash-based ID is content-derived and round-trip-stable, which downstream consumers (Plan 02 Recommendation.external_resource_chunk_id, Plan 03 verify_resource's external_resource lookup) depend on for cross-run identity.
- SHA-1 truncated to 8 hex chars yields ~4 billion bucket IDs — collision probability across a per-gear chunks.jsonl of <10k external_resource chunks is negligible (birthday-paradox floor at >65k entries before any 50% collision risk).
- Collision behavior: if two distinct canonical URLs DO hash to the same 8-char prefix, the sweep emits both as separate chunks (sweep does not re-key on collision); Plan 02's aggregator groups by canonical URL not by chunk ID, so the duplicate IDs do not corrupt the recommendation surface. A v2.x hardening could lengthen the hash to 12 chars if collisions are ever observed.

    """canonicalize_url — collapse URL variants so two chunks pointing at the
    same resource count as one citation, not two. Library-light pure function;
    no requests / no network. T-04-01 mitigation: rejects non-http(s) schemes
    (returns "" so callers can filter)."""
    from __future__ import annotations
    from urllib.parse import urlparse, urlunparse, parse_qsl, urlencode

    # Tracking params to strip across ALL hosts. Source: locked at v2.0 per
    # CITATION-04 + spike findings. Keep this list small — adding a host-
    # specific param means a host-specific case below, not a new entry here.
    TRACKING_PARAMS = frozenset({
        "si",                            # YouTube share-tracking
        "utm_source", "utm_medium", "utm_campaign", "utm_term", "utm_content",
        "feature",                       # YouTube share-feature attribution
        "fbclid", "gclid", "mc_cid", "mc_eid",  # common ad/email trackers
    })

    YOUTUBE_HOSTS = frozenset({"www.youtube.com", "youtube.com", "m.youtube.com"})

    def canonicalize_url(url: str) -> str:
        """Return the canonical form of `url`, or "" if `url` is unusable.

        Rules (locked at v2.0):
          - Reject non-http(s) schemes → "".
          - Lowercase scheme + host.
          - youtu.be/<id>  ↔  www.youtube.com/watch?v=<id>  (canonical = the watch form).
          - Strip TRACKING_PARAMS from the query.
          - Strip a single trailing slash from a non-root path.
          - Preserve all other query params and their order (sort for determinism).
        """
        if not isinstance(url, str) or not url.strip():
            return ""
        try:
            p = urlparse(url.strip())
        except (ValueError, TypeError):
            return ""
        scheme = (p.scheme or "").lower()
        if scheme not in ("http", "https"):
            return ""
        host = (p.hostname or "").lower()
        if not host:
            return ""
        path = p.path or ""
        query_pairs = parse_qsl(p.query, keep_blank_values=False)

        # YouTube short → long canonicalization.
        if host == "youtu.be" and path and path != "/":
            video_id = path.strip("/").split("/")[0]
            if video_id:
                host = "www.youtube.com"
                path = "/watch"
                # Drop any existing v=, prepend our extracted one.
                query_pairs = [("v", video_id)] + [
                    (k, v) for k, v in query_pairs if k != "v"
                ]
        elif host in YOUTUBE_HOSTS:
            # Normalize host to www.youtube.com for the watch URL form.
            host = "www.youtube.com"

        # Strip tracking params.
        query_pairs = [(k, v) for k, v in query_pairs if k not in TRACKING_PARAMS]
        # Deterministic ordering.
        query_pairs.sort()
        query = urlencode(query_pairs)

        # Strip a single trailing slash from non-root paths.
        if path.endswith("/") and path != "/":
            path = path.rstrip("/")

        # Rebuild. Drop fragment (collapses #section variants; deep_link lives
        # in provenance, not in the canonical citation key).
        return urlunparse((scheme, host, path, "", query, ""))
    ```

    """external_resource_sweep — post-write idempotent sweep that guarantees
    CITATION-01 (every URL has an external_resource chunk with populated
    citing_chunk_ids) and CITATION-04 (URLs in external_resource chunks are
    canonical).

    Called by write_chunks at the end of every append. Reads chunks.jsonl
    atomically, computes the citation graph, writes back via tempfile +
    os.replace."""
    from __future__ import annotations
    import hashlib
    import json
    import os
    import re
    import tempfile
    from datetime import datetime, timezone
    from pathlib import Path
    from typing import Any, Iterable, Optional

    try:
        from canonicalize_url import canonicalize_url
    except ImportError:  # pragma: no cover — package-relative fallback
        from .canonicalize_url import canonicalize_url  # type: ignore

    URL_RE = re.compile(r"https?://[^\s<>\"'\\)]+", re.IGNORECASE)

    def _classify_resource_type(url: str) -> str:
        """Sweep's AUTHORITATIVE resource_type classifier. Overrides any value
        set by a source-class parser on merge (reddit.py emits 'article' for
        reddit-post URLs; this function emits 'reddit-post' and the sweep
        overwrites). Locked at v2.0; a future refactor could extract a shared
        classify_url helper used by both this module and reddit.py."""
        u = url.lower()
        if "youtube.com/watch" in u or "youtu.be/" in u:
            return "youtube"
        if "reddit.com/r/" in u and "/comments/" in u:
            return "reddit-post"
        if u.endswith((".png", ".jpg", ".jpeg", ".gif", ".webp")):
            return "image"
        return "article"

    def _sweep_id(canonical_url: str) -> str:
        """Stable, content-derived id for a sweep-emitted external_resource.
        SHA-1 truncated to 8 hex chars — same canonical URL ALWAYS yields the
        same id across re-runs (W5 mitigation: a monotonic counter would
        re-issue the same id for a DIFFERENT URL when discovery order changes;
        a hash-based id is round-trip-stable, which Plan 02's
        Recommendation.external_resource_chunk_id depends on)."""
        return "ext-sweep-" + hashlib.sha1(
            canonical_url.encode("utf-8")
        ).hexdigest()[:8]

    def _walk_strings(value: Any) -> Iterable[str]:
        if isinstance(value, str):
            yield value
        elif isinstance(value, dict):
            for v in value.values():
                yield from _walk_strings(v)
        elif isinstance(value, (list, tuple)):
            for v in value:
                yield from _walk_strings(v)

    def extract_urls_from_chunk(chunk: dict) -> list[str]:
        """Return canonical URLs referenced by a non-external_resource chunk.
        Sources: chunk["content"] (recursive), provenance.url, provenance.deep_link.
        For external_resource chunks: returns [] (handled separately in the sweep)."""
        if chunk.get("type") == "external_resource":
            return []
        urls: list[str] = []
        for s in _walk_strings(chunk.get("content")):
            urls.extend(URL_RE.findall(s))
        prov = chunk.get("provenance") or {}
        for key in ("url", "deep_link"):
            val = prov.get(key)
            if isinstance(val, str) and val:
                urls.append(val)
        out: list[str] = []
        seen: set[str] = set()
        for u in urls:
            c = canonicalize_url(u.rstrip(".,;:!?)"))
            if c and c not in seen:
                seen.add(c)
                out.append(c)
        return out

    def ensure_external_resource_chunks(
        chunks_jsonl_path,
        gear_root: Optional[str] = None,
    ) -> None:
        """Idempotent post-write sweep. Reads chunks.jsonl, builds the citation
        graph keyed by canonical URL, rewrites chunks.jsonl atomically.

        Signature mirrors write_chunks: path FIRST (positional), gear_root
        keyword with default. Sweep currently does not use gear_root (no
        path-containment check at this layer — write_chunks already validated
        the path before invoking the sweep) but the param is accepted for
        signature parity and future hardening."""
        chunks_path = Path(chunks_jsonl_path)
        if not chunks_path.exists():
            return
        rows: list[dict] = []
        with chunks_path.open("r", encoding="utf-8") as fp:
            for line in fp:
                line = line.strip()
                if not line:
                    continue
                try:
                    rows.append(json.loads(line))
                except json.JSONDecodeError:
                    # Preserve malformed lines verbatim by skipping the parse;
                    # safer than silently dropping. Plan 03-01 _read_existing
                    # raises on malformed lines, so this branch should not
                    # trigger in production — kept as defense-in-depth.
                    continue

        # Build citation graph from non-external_resource chunks.
        url_to_citing: dict[str, set[str]] = {}
        for chunk in rows:
            if chunk.get("type") == "external_resource":
                continue
            chunk_id = chunk.get("id")
            if not chunk_id:
                continue
            for url in extract_urls_from_chunk(chunk):
                url_to_citing.setdefault(url, set()).add(chunk_id)

        # Canonicalize + merge existing external_resource chunks.
        # NB: cross_source_match_candidates is NAME-based not id-based (verified
        # against write_chunk.compute_cross_source_matches L135-L186), so
        # dropping a duplicate external_resource by id is safe — no other
        # chunk's match-candidates field references the dropped id.
        existing: dict[str, dict] = {}  # canonical_url -> chunk dict
        kept_non_ext: list[dict] = []
        for chunk in rows:
            if chunk.get("type") != "external_resource":
                kept_non_ext.append(chunk)
                continue
            content = chunk.get("content") or {}
            raw_url = content.get("url", "") or ""
            canonical = canonicalize_url(raw_url)
            if not canonical:
                # Unusable URL — drop the chunk (it cannot be cited).
                continue
            if canonical in existing:
                # Duplicate external_resource for the same canonical URL.
                # Merge citing_chunk_ids; keep the FIRST occurrence's id and
                # remaining fields (preserves Equipboard's parent backlink).
                prior = existing[canonical]
                prior_cids = set(prior["content"].get("citing_chunk_ids") or [])
                new_cids = set(content.get("citing_chunk_ids") or [])
                merged = sorted(prior_cids | new_cids)
                prior["content"]["citing_chunk_ids"] = merged
                continue
            # Write canonical URL back to the chunk.
            content["url"] = canonical
            # AUTHORITATIVE override of resource_type — the sweep's classifier
            # wins on merge (reddit.py emits "article" for reddit-post URLs;
            # the sweep emits "reddit-post" and overwrites here).
            content["resource_type"] = _classify_resource_type(canonical)
            chunk["content"] = content
            existing[canonical] = chunk

        # Update citing_chunk_ids on existing external_resource chunks.
        for url, chunk in existing.items():
            prior = set(chunk["content"].get("citing_chunk_ids") or [])
            discovered = url_to_citing.get(url, set())
            chunk["content"]["citing_chunk_ids"] = sorted(prior | discovered)

        # Append new external_resource chunks for canonical URLs without one.
        now_iso = datetime.now(timezone.utc).isoformat().replace("+00:00", "Z")
        new_ext: list[dict] = []
        for url, citing in url_to_citing.items():
            if url in existing:
                continue
            new_ext.append({
                "id": _sweep_id(url),
                "type": "external_resource",
                "source": "sweep",
                "content": {
                    "resource_type": _classify_resource_type(url),
                    "creator": "",
                    "title": "",
                    "url": url,
                    "updated": None,
                    "relevance": "",
                    "citing_chunk_ids": sorted(citing),
                },
                # tier_used = None: the sweep does NOT fetch — this chunk is
                # synthetic, derived from URLs in OTHER chunks. The spike-
                # findings tier ladder reserves tier_used=0 for "manual user-
                # paste (escape hatch)" which is a DIFFERENT semantic.
                # Downstream queries treat null as "did not fetch."
                "tier_used": None,
                "provenance": {
                    "url": url,
                    "scraped_at": now_iso,
                },
            })

        # Sort new_ext by id so the file is deterministically ordered across
        # re-runs (hash-based ids give us natural cross-run stability, but
        # sorting also prevents dict-iteration-order differences from
        # surfacing as byte-level diffs and breaking idempotency tests).
        new_ext.sort(key=lambda c: c["id"])

        # Rewrite atomically: non-ext chunks (in original order) + existing ext
        # chunks (in original canonical-URL insertion order) + new ext chunks
        # (id-sorted).
        all_rows = (
            kept_non_ext
            + [existing[u] for u in existing]
            + new_ext
        )
        fd, tmp_path = tempfile.mkstemp(
            prefix=".chunks-sweep-", dir=str(chunks_path.parent)
        )
        try:
            with os.fdopen(fd, "w", encoding="utf-8") as tmp_fp:
                for row in all_rows:
                    tmp_fp.write(json.dumps(row, ensure_ascii=False) + "\n")
            os.replace(tmp_path, chunks_path)
        except Exception:
            try:
                os.unlink(tmp_path)
            except OSError:
                pass
            raise
    ```
